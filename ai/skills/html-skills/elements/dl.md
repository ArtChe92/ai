# dl

## Сценарии использования

Список определений — пары "термин — описание".

**Когда выбрать другой элемент:**

- Простой список без определений → `ul` или `ol`
- Таблица с данными → `table`

**Примеры использования:**

- Глоссарии
- Метаданные (автор, дата, категория)
- FAQ (вопрос — ответ)
- Характеристики товара

## Связанные элементы

| Элемент | Связь                  |
| ------- | ---------------------- |
| `dt`    | Обязательно — термин   |
| `dd`    | Обязательно — описание |

## Необходимые атрибуты

Специфичных атрибутов нет.

## Визуальные состояния

Не применимо.

## Правила доступности

- Структура: `dl` содержит пары `dt` + `dd`
- Один `dt` может иметь несколько `dd`
- Несколько `dt` могут иметь один `dd`
- Можно группировать в `div` для стилизации (HTML5)

## SEO

Помогает поисковикам понять структурированные данные (FAQ, характеристики).

## Пример

```tsx
{
    /* Простой список определений */
}
<dl>
    <dt>HTML</dt>
    <dd>Язык разметки гипертекста</dd>

    <dt>CSS</dt>
    <dd>Каскадные таблицы стилей</dd>
</dl>;

{
    /* С группировкой для стилизации */
}
<dl className="meta">
    <div className="meta-item">
        <dt>HTML</dt>
        <dd>Язык разметки гипертекста</dd>
    </div>
    <div className="meta-item">
        <dt>CSS</dt>
        <dd>Каскадные таблицы стилей</dd>
    </div>
</dl>;
```
