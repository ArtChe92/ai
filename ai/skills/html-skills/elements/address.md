# address

## Сценарии использования

Контактная информация автора или владельца документа/статьи.

**Когда выбрать другой элемент:**

- Произвольный почтовый адрес в тексте → `p`
- Адрес доставки в форме → `p` или `fieldset`

**Примеры использования:**

- Контакты автора статьи
- Контакты компании в футере
- Email, телефон, ссылки на соцсети

**Правило:** `address` — не для любого адреса, а для контактов владельца контента.

## Связанные элементы

| Элемент   | Связь                               |
| --------- | ----------------------------------- |
| `footer`  | Родитель — часто в подвале          |
| `article` | Родитель — контакты автора статьи   |
| `a`       | Дочерний — ссылки на email, соцсети |

> См. footer.md, article.md

## Необходимые атрибуты

Специфичных атрибутов нет.

## Визуальные состояния

Не применимо.

## Правила доступности

- Не вкладывать `header`, `footer`, `article`, `section`, `address` внутрь `address`
- Не использовать для произвольных адресов — только контакты автора/владельца
- Screen reader объявляет как контактную информацию

## SEO

Помогает поисковикам определить контактную информацию сайта/автора.

## Пример

```tsx
{
    /* В футере сайта */
}
<footer>
    <address>
        <a href="mailto:info@example.com">info@example.com</a>
        <a href="tel:+71234567890">+7 (123) 456-78-90</a>
    </address>
</footer>;

{
    /* В статье */
}
<article>
    <h1>Заголовок статьи</h1>
    <p>Содержимое...</p>

    <footer>
        <address>
            Автор: <a href="/authors/1">Имя автора</a>
        </address>
    </footer>
</article>;
```
