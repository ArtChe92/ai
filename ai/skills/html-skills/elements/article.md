# article

## Сценарии использования

Самостоятельный, независимый контент — может быть переиспользован отдельно.

**Когда выбрать другой элемент:**

- Контент зависит от контекста страницы → `section`
- Просто группировка для стилей → `div`

**Примеры использования:**

- Пост в блоге
- Новость
- Карточка товара
- Комментарий
- Виджет

**Правило:** если контент имеет смысл вне страницы (в RSS, при шаринге) → `article`.

## Связанные элементы

| Элемент   | Связь                                     |
| --------- | ----------------------------------------- |
| `header`  | Дочерний — заголовок статьи               |
| `footer`  | Дочерний — мета-информация                |
| `h1`-`h6` | Обязательно — заголовок                   |
| `time`    | Дочерний — дата публикации                |
| `section` | Дочерний — разделы внутри статьи          |
| `aside`   | Дочерний — дополнительная информация      |
| `article` | Дочерний — вложенные статьи (комментарии) |

> См. header.md, footer.md, section.md, aside.md, time.md

## Необходимые атрибуты

### По ситуации

| Атрибут           | Когда                     |
| ----------------- | ------------------------- |
| `aria-labelledby` | Связь с заголовком статьи |

## Визуальные состояния

Не применимо.

## Правила доступности

- Каждый `article` должен иметь заголовок (`h1`-`h6`)
- Вложенные `article` допустимы (комментарии к посту)
- Screen reader объявляет как "article"

## SEO

- Помогает поисковикам выделить самостоятельный контент
- Улучшает понимание структуры для rich snippets

## Пример

```tsx
<article>
    <header>
        <h2>Заголовок статьи</h2>
        <time dateTime="2026-01-27">27 января 2026</time>
    </header>

    <p>Содержимое статьи...</p>

    <footer>
        <address>
            Автор: <a href="/authors/1">Имя автора</a>
        </address>
    </footer>
</article>
```
