# h1-h6

## Сценарии использования

Заголовки — определяют иерархию и структуру контента.

**Когда выбрать другой элемент:**

- Визуально крупный текст без смысловой иерархии → `p` + стили
- Подпись к изображению → `figcaption`

**Уровни:**

| Тег  | Использование              |
| ---- | -------------------------- |
| `h1` | Главный заголовок страницы |
| `h2` | Основные разделы           |
| `h3` | Подразделы внутри h2       |
| `h4` | Подразделы внутри h3       |
| `h5` | Редко используется         |
| `h6` | Редко используется         |

## Связанные элементы

| Элемент   | Связь                                    |
| --------- | ---------------------------------------- |
| `section` | Родитель — каждая секция имеет заголовок |
| `article` | Родитель — статья имеет заголовок        |
| `header`  | Родитель — заголовок в шапке             |
| `hgroup`  | Группировка заголовка с подзаголовком    |

> См. section.md, article.md

## Необходимые атрибуты

### По ситуации

| Атрибут | Когда                                    |
| ------- | ---------------------------------------- |
| `id`    | Для якорных ссылок или `aria-labelledby` |

## Визуальные состояния

Не применимо.

## Правила доступности

- Один `h1` на страницу
- Не пропускать уровни (h1 → h3 — неправильно)
- Заголовок для стилей, не для семантики → использовать `p` или `span`
- Screen reader использует заголовки для навигации

## SEO

- `h1` — главный сигнал темы страницы, один на страницу
- Заголовки влияют на ранжирование
- Ключевые слова в заголовках важны
- Иерархия должна быть логичной

```tsx
    <section>
        <h2>Основы</h2>

        <section>
            <h3>Компоненты</h3>
            <p>...</p>
        </section>

        <section>
            <h3>Пропсы</h3>
            <p>...</p>
        </section>
    </section>

    <section>
        <h2>Продвинутые темы</h2>

        <section>
            <h3>Хуки</h3>
            <p>...</p>
        </section>
    </section>
</main>
```
