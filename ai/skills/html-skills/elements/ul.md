# ul

## Сценарии использования

Неупорядоченный список — порядок элементов не важен.

**Когда выбрать другой элемент:**

- Порядок важен (инструкции, рейтинг) → `ol`
- Пары "термин — определение" → `dl`
- Навигационное меню → `ul` внутри `nav`

## Связанные элементы

| Элемент | Связь                  |
| ------- | ---------------------- |
| `li`    | Обязательно — дочерний элемент |
| `nav`   | Родитель для навигации |

> См. li.md, ol.md, nav.md

## Необходимые атрибуты

### По ситуации

| Атрибут     | Когда                                  |
| ----------- | -------------------------------------- |
| `role="list"` | Если CSS сбрасывает `list-style: none` (Safari) |
| `aria-label` | Если список требует пояснения          |

⚠️ Safari удаляет семантику списка при `list-style: none`. Добавлять `role="list"` если стили сброшены.

## Визуальные состояния

Не применимо.

## Правила доступности

- Внутри `ul` только `li` (напрямую)
- При сбросе стилей → `role="list"` на `ul`, `role="listitem"` на `li`
- Вложенные списки — внутри `li`, не рядом

## SEO

Не влияет напрямую. Но структурированные списки улучшают читаемость для поисковиков.

## Пример

```tsx
<ul role="list">
    <li>Первый пункт</li>
    <li>Второй пункт</li>
    <li>
        Третий пункт
        <ul role="list">
            <li>Вложенный пункт</li>
        </ul>
    </li>
</ul>
```
