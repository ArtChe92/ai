# footer

## Сценарии использования

Завершающая часть страницы или секции.

**Когда выбрать другой элемент:**

- Просто обёртка для стилей → `div`

**Примеры использования:**

- Подвал сайта (копирайт, контакты, ссылки)
- Мета-информация в конце `article` (автор, дата, теги)

## Связанные элементы

| Элемент   | Связь                                |
| --------- | ------------------------------------ |
| `nav`     | Навигация в подвале                  |
| `address` | Контактная информация                |
| `article` | Родитель — статья с мета-информацией |

> См. nav.md, address.md

## Необходимые атрибуты

### По ситуации

| Атрибут              | Когда                                                         |
| -------------------- | ------------------------------------------------------------- |
| `role="contentinfo"` | Только для основного `footer` страницы (обычно автоматически) |

⚠️ `footer` внутри `article` или `section` не получает роль `contentinfo` — это правильно.

## Визуальные состояния

Не применимо.

## Правила доступности

- Основной `footer` страницы — один, имеет роль `contentinfo`
- `footer` внутри секций — не имеет этой роли
- Не вкладывать `footer` в `header`, `footer`, `address`

## SEO

Помогает поисковикам определить служебную информацию сайта.

## Пример

```tsx
<footer>
    <nav aria-label="Навигация в подвале">
        <ul role="list">
            <li>
                <a href="/privacy">Политика конфиденциальности</a>
            </li>
            <li>
                <a href="/terms">Условия использования</a>
            </li>
        </ul>
    </nav>

    <address>
        <a href="mailto:info@example.com">info@example.com</a>
    </address>

    <p>© 2026 Company Name</p>
</footer>
```
