# nav

## Сценарии использования

Контейнер для навигационных ссылок.

**Когда выбрать другой элемент:**

- Одиночная ссылка → `a`
- Ссылки не для навигации (соцсети, партнёры) → `ul` без `nav`

**Примеры использования:**

- Главное меню
- Хлебные крошки
- Пагинация
- Оглавление
- Боковое меню

## Связанные элементы

| Элемент  | Связь                         |
| -------- | ----------------------------- |
| `ul`     | Список ссылок внутри          |
| `a`      | Ссылки навигации              |
| `header` | Часто содержит основной `nav` |

> См. a.md, ul.md, header.md

## Необходимые атрибуты

### По ситуации

| Атрибут               | Когда                                   |
| --------------------- | --------------------------------------- |
| `aria-label`          | Несколько `nav` на странице — различать |
| `aria-labelledby`     | Есть видимый заголовок навигации        |
| `aria-current="page"` | На ссылке текущей страницы              |

⚠️ Если на странице несколько `nav`, каждый должен иметь уникальный `aria-label`.

## Визуальные состояния

Не применимо к самому `nav`. Состояния — на ссылках внутри.

## Правила доступности

- Несколько `nav` → уникальные `aria-label` ("Главное меню", "Подвал", "Хлебные крошки")
- Текущая страница → `aria-current="page"` на ссылке
- Мобильное меню → `aria-expanded` на кнопке-бургере
- Не оборачивать в `nav` все подряд ссылки

## SEO

Помогает поисковикам понять структуру навигации сайта.

## Пример

```tsx
<nav aria-label="Главное меню">
    <ul role="list">
        <li>
            <a href="/" aria-current="page">
                Главная
            </a>
        </li>
        <li>
            <a href="/catalog">Каталог</a>
        </li>
        <li>
            <a href="/about">О нас</a>
        </li>
    </ul>
</nav>
```
