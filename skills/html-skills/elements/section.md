# section

## Сценарии использования

Тематический раздел страницы — группа связанного контента.

**Когда выбрать другой элемент:**

- Контент самостоятельный, независимый → `article`
- Просто обёртка для стилей → `div`
- Боковая информация → `aside`

**Примеры использования:**

- Раздел "О нас" на лендинге
- Глава документа
- Табы (каждая вкладка — section)
- Секции внутри статьи

**Правило:** если нет подходящего заголовка для блока — скорее всего, это не `section`.

## Связанные элементы

| Элемент   | Связь                          |
| --------- | ------------------------------ |
| `h1`-`h6` | Обязательно — заголовок секции |
| `header`  | Дочерний — вводная часть       |
| `article` | Родитель или дочерний          |
| `aside`   | Сосед — дополнительный контент |
| `main`    | Родитель                       |

> См. headings.md, article.md, aside.md, main.md

## Необходимые атрибуты

### По ситуации

| Атрибут           | Когда                                      |
| ----------------- | ------------------------------------------ |
| `aria-labelledby` | Связь с заголовком секции                  |
| `aria-label`      | Если нет видимого заголовка (нежелательно) |

## Визуальные состояния

Не применимо.

## Правила доступности

- Каждый `section` должен иметь заголовок (`h1`-`h6`)
- Без заголовка `section` не имеет смысла для screen reader

## SEO

Помогает поисковикам понять структуру и тематику разделов страницы.

## Пример

```tsx
<main>
    <section aria-labelledby="about-title">
        <h2 id="about-title">О компании</h2>
        <p>Описание компании...</p>
    </section>

    <section aria-labelledby="services-title">
        <h2 id="services-title">Услуги</h2>
        <ul>
            <li>Разработка</li>
            <li>Дизайн</li>
        </ul>
    </section>
</main>
```
