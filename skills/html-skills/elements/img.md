# img

## Сценарии использования

Изображение с информационным содержанием.

**Когда выбрать другой элемент:**

- Адаптивное изображение (разные размеры/форматы) → `picture`
- Декоративное изображение → CSS `background-image`
- Иконка в кнопке/ссылке → SVG inline или `background-image`
- Сложная графика с интерактивностью → `svg`

## Связанные элементы

| Элемент      | Связь                               |
| ------------ | ----------------------------------- |
| `figure`     | Родитель — изображение с подписью   |
| `figcaption` | Сосед — подпись к изображению       |
| `picture`    | Родитель — адаптивные изображения   |
| `a`          | Родитель — кликабельное изображение |

> См. picture.md

## Необходимые атрибуты

### Всегда

| Атрибут | Назначение                              |
| ------- | --------------------------------------- |
| `src`   | Путь к изображению                      |
| `alt`   | Альтернативный текст — всегда указывать |

### По ситуации

| Атрибут                | Когда                                          |
| ---------------------- | ---------------------------------------------- |
| `width` / `height`     | Всегда желательно — предотвращает сдвиг layout |
| `loading="lazy"`       | Изображения ниже первого экрана                |
| `decoding="async"`     | Неблокирующее декодирование                    |
| `srcset`               | Адаптивные размеры                             |
| `sizes`                | Размеры для разных viewport                    |
| `fetchpriority="high"` | LCP-изображение (главное на экране)            |

### Правила для alt

| Ситуация                  | Значение alt                        |
| ------------------------- | ----------------------------------- |
| Информативное изображение | Описание содержимого                |
| Декоративное изображение  | `alt=""` (пустой, но обязателен)    |
| Изображение-ссылка        | Описание действия/назначения ссылки |
| Изображение с текстом     | Текст с изображения                 |
| Сложная инфографика       | Краткое описание + ссылка на детали |

⚠️ Не начинать alt с "Изображение...", "Картинка..." — screen reader уже объявляет это.

## Визуальные состояния

| Состояние | Описание                             |
| --------- | ------------------------------------ |
| `loading` | Плейсхолдер или скелетон до загрузки |
| `error`   | Fallback при ошибке загрузки         |

## Правила доступности

- `alt` обязателен всегда, даже если пустой
- Декоративные изображения → `alt=""`
- Сложные изображения → `aria-describedby` с развёрнутым описанием
- Изображение внутри ссылки без текста → `alt` описывает назначение ссылки

## SEO

- `alt` индексируется поисковиками
- Информативный `alt` с ключевыми словами (без спама)
- Имя файла тоже учитывается — осмысленные имена лучше
- `width` / `height` улучшают Core Web Vitals (CLS)

## Пример

```tsx
{
    /* Информативное изображение */
}
<img
    src="/images/team.jpg"
    alt="Команда разработчиков на конференции 2025"
    width={800}
    height={600}
    loading="lazy"
    decoding="async"
/>;

{
    /* Декоративное изображение */
}
<img src="/images/divider.svg" alt="" aria-hidden="true" />;

{
    /* Изображение с подписью */
}
<figure>
    <img src="/images/chart.png" alt="График роста продаж за 2025 год" width={600} height={400} />
    <figcaption>Рост продаж составил 45% по сравнению с 2024</figcaption>
</figure>;
```
