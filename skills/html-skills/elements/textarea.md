# textarea

## Сценарии использования

Многострочный ввод текста.

**Когда выбрать другой элемент:**

- Однострочный ввод → `input`
- Форматированный текст (WYSIWYG) → `div` с `contenteditable` или специализированный редактор

**Примеры использования:**

- Комментарии
- Сообщения в форме обратной связи
- Описание

## Связанные элементы

| Элемент    | Связь                       |
| ---------- | --------------------------- |
| `label`    | Обязательно — подпись поля  |
| `form`     | Родительский контейнер      |
| `fieldset` | Группировка связанных полей |

> См. label.md, form.md

## Необходимые атрибуты

### Всегда

| Атрибут | Назначение                    |
| ------- | ----------------------------- |
| `id`    | Связь с label через `htmlFor` |
| `name`  | Идентификатор в данных формы  |

### По ситуации

| Атрибут            | Когда                                 |
| ------------------ | ------------------------------------- |
| `rows`             | Начальная высота в строках            |
| `cols`             | Начальная ширина в символах           |
| `required`         | Обязательное поле                     |
| `disabled`         | Поле неактивно                        |
| `readonly`         | Только для чтения                     |
| `placeholder`      | Подсказка формата (не замена label!)  |
| `maxlength`        | Максимальное количество символов      |
| `minlength`        | Минимальное количество символов       |
| `autocomplete`     | Подсказки браузера                    |
| `spellcheck`       | Проверка орфографии                   |
| `wrap`             | `soft` / `hard` — перенос строк       |
| `aria-describedby` | Связь с описанием ошибки / подсказкой |
| `aria-invalid`     | Сигнал об ошибке для screen reader    |

⚠️ Размеры лучше задавать через CSS (`width`, `min-height`), а не `rows`/`cols`.

## Визуальные состояния

| Состояние  | Описание                                             |
| ---------- | ---------------------------------------------------- |
| `default`  | Базовый вид                                          |
| `hover`    | Наведение курсора                                    |
| `focus`    | Фокус с клавиатуры — обязательно видимый             |
| `disabled` | Неактивное — сниженный контраст, cursor: not-allowed |
| `readonly` | Только чтение — отличать от disabled                 |
| `error`    | Ошибка валидации — border, иконка, цвет              |

## Правила доступности

- Всегда связывать с `label` через `id` + `htmlFor`
- `placeholder` — не замена label
- Ошибки связывать через `aria-describedby`
- При ошибке добавлять `aria-invalid="true"`
- Счётчик символов — связать через `aria-describedby`

## SEO

Не влияет напрямую.

## Пример

```tsx
<div className="field">
    <label htmlFor="message">
        Сообщение <span aria-hidden="true">*</span>
    </label>

    <textarea
        id="message"
        name="message"
        required
        rows={5}
        maxLength={1000}
        aria-describedby={error ? 'message-error' : 'message-hint'}
        aria-invalid={error ? 'true' : undefined}
    />

    <span id="message-hint" className="hint">
        Максимум 1000 символов
    </span>

    {error && (
        <span id="message-error" role="alert" className="error">
            {error}
        </span>
    )}
</div>
```
