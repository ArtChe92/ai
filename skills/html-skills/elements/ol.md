# ol

## Сценарии использования

Упорядоченный список — порядок элементов важен.

**Когда выбрать другой элемент:**

- Порядок не важен → `ul`
- Пары "термин — определение" → `dl`

**Примеры использования:**

- Пошаговые инструкции
- Рейтинги, топы
- Юридические пункты

## Связанные элементы

| Элемент | Связь                          |
| ------- | ------------------------------ |
| `li`    | Обязательно — дочерний элемент |

> См. li.md, ul.md

## Необходимые атрибуты

### По ситуации

| Атрибут       | Когда                                  |
| ------------- | -------------------------------------- |
| `start`       | Начать нумерацию не с 1                |
| `reversed`    | Обратный порядок нумерации             |
| `type`        | Тип маркера (1, a, A, i, I)            |
| `role="list"` | Если CSS сбрасывает `list-style: none` |

## Визуальные состояния

Не применимо.

## Правила доступности

- Внутри `ol` только `li` (напрямую)
- При сбросе стилей → `role="list"` на `ol`, `role="listitem"` на `li`
- Вложенные списки — внутри `li`, не рядом

## SEO

Упорядоченные списки помогают поисковикам понять структуру инструкций и рейтингов.

## Пример

```tsx
<ol>
    <li>Открыть терминал</li>
    <li>
        Выполнить команду <code>npm install</code>
    </li>
    <li>Дождаться завершения установки</li>
</ol>
```
