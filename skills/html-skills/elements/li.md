# li

## Сценарии использования

Элемент списка — пункт внутри `ul` или `ol`.

**Когда выбрать другой элемент:**

- Определение термина → `dd` внутри `dl`

## Связанные элементы

| Элемент | Связь              |
| ------- | ------------------ |
| `ul`    | Родитель — неупорядоченный список |
| `ol`    | Родитель — упорядоченный список   |

> См. ul.md, ol.md

## Необходимые атрибуты

### По ситуации

| Атрибут         | Когда                                      |
| --------------- | ------------------------------------------ |
| `value`         | Переопределить номер в `ol`                |
| `role="listitem"` | Если на родителе сброшены стили          |

## Визуальные состояния

Не применимо (если `li` не интерактивный).

Для интерактивных списков (выбор, меню):

| Состояние  | Описание                  |
| ---------- | ------------------------- |
| `hover`    | Наведение курсора         |
| `focus`    | Фокус с клавиатуры        |
| `selected` | Выбранный элемент         |

## Правила доступности

- `li` только внутри `ul` или `ol`
- Вложенный список — внутри `li`, не после него
- При сбросе стилей родителя → `role="listitem"`

## SEO

Не влияет напрямую.

## Пример

```tsx
<ul role="list">
    <li role="listitem">Простой пункт</li>
    <li role="listitem">
        Пункт с вложенным списком
        <ul role="list">
            <li role="listitem">Вложенный пункт</li>
        </ul>
    </li>
</ul>
```
