# header

## Сценарии использования

Вводная часть страницы или секции.

**Когда выбрать другой элемент:**

- Просто обёртка для стилей → `div`

**Примеры использования:**

- Шапка сайта (лого, навигация, поиск)
- Заголовочная часть `article` или `section`

## Связанные элементы

| Элемент   | Связь                          |
| --------- | ------------------------------ |
| `nav`     | Навигация внутри шапки         |
| `search`  | Поиск внутри шапки             |
| `h1`-`h6` | Заголовок                      |
| `section` | Родитель — секция с заголовком |
| `article` | Родитель — статья с заголовком |

> См. nav.md, search.md, headings.md

## Необходимые атрибуты

### По ситуации

| Атрибут         | Когда                                                         |
| --------------- | ------------------------------------------------------------- |
| `role="banner"` | Только для основного `header` страницы (обычно автоматически) |

⚠️ `header` внутри `article` или `section` не получает роль `banner` автоматически — это правильно.

## Визуальные состояния

Не применимо.

## Правила доступности

- Основной `header` страницы — один, имеет роль `banner`
- `header` внутри секций — не имеет роли `banner`
- Не вкладывать `header` в `header`, `footer`, `address`

## SEO

Помогает поисковикам определить структуру страницы.

## Пример

```tsx
<header>
    <a href="/" aria-label="На главную">
        <Logo aria-hidden="true" />
    </a>

    <nav aria-label="Главное меню">
        <ul role="list">
            <li>
                <a href="/catalog">Каталог</a>
            </li>
            <li>
                <a href="/about">О нас</a>
            </li>
        </ul>
    </nav>
</header>
```
