# search

## Сценарии использования

Контейнер для поиска — форма поиска или фильтры.

**Когда выбрать другой элемент:**

- Обычная форма без поиска → `form`
- Навигация → `nav`

**Примеры использования:**

- Поиск по сайту
- Фильтрация списка
- Поиск в таблице

## Связанные элементы

| Элемент  | Связь                    |
| -------- | ------------------------ |
| `form`   | Дочерний — форма поиска  |
| `input`  | Дочерний — поле ввода    |
| `button` | Дочерний — кнопка поиска |

> См. form.md, input.md

## Необходимые атрибуты

### По ситуации

| Атрибут           | Когда                                      |
| ----------------- | ------------------------------------------ |
| `aria-label`      | Несколько `search` на странице — различать |
| `aria-labelledby` | Есть видимый заголовок поиска              |

⚠️ Если на странице несколько блоков поиска, каждый должен иметь уникальный `aria-label`.

## Визуальные состояния

Не применимо к самому `search`. Состояния — на элементах внутри.

## Правила доступности

- Автоматически получает роль `search` — не нужно добавлять `role="search"`
- Несколько `search` на странице → уникальные `aria-label`
- Заменяет устаревший паттерн `<form role="search">`

## SEO

Помогает поисковикам определить функционал поиска на сайте.

## Пример

```tsx
{/* Базовый поиск */}
<search>
    <form>
        <label htmlFor="search-input">
            Поиск по сайту
        </label>
        <input id="search-input" type="search">
        <button type="submit">
            <SearchIcon aria-hidden="true" />
            <span>Искать</span>
        </button>
    </form>
</search>

{/* Несколько поисков на странице */}
<header>
    <search aria-label="Поиск по сайту">
        <form>...</form>
    </search>
</header>

<main>
    <search aria-label="Фильтр товаров">
        <form>...</form>
    </search>
</main>
```
